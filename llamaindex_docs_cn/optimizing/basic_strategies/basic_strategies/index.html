基础优化策略#

当需要快速提升性能并优化 RAG 工作流时，有许多简单的方法可以尝试。

提示工程#

如果遇到与 LLM 相关的故障，如幻觉或输出格式问题，这应该是首要尝试的解决方案。

以下列出从简单到进阶的任务：

  1. 检查 RAG 工作流中使用的提示（例如问答提示）并进行定制
  2. 定制提示

  3. 高级提示

  4. 尝试添加提示函数，实现动态注入少量示例或处理输入内容

  5. 高级提示

嵌入模型#

选择合适的嵌入模型对整体性能影响重大： - 可能需要比 OpenAI 默认的 text-embedding-ada-002 更优的模型？ - 需要扩展到本地服务器部署？ - 需要针对特定语言优化的嵌入模型？

除 OpenAI 外，还有多种嵌入 API 选择，包括本地运行嵌入模型或自建服务器。

MTEB 排行榜是评估当前最佳嵌入模型的权威资源，基于 50+ 数据集和任务进行排名。

注意： 与可随时更换的 LLM 不同，更换嵌入模型后必须重新索引数据，且需确保索引和查询使用相同模型。

我们提供所有支持的嵌入模型集成列表。

分块大小#

根据数据类型或检索结果，可能需要调整分块大小或重叠量：

文档被索引时会按指定重叠量分块。默认分块大小为 1024，重叠量为 20。

调整这些参数将改变生成的嵌入向量。较小分块使嵌入更精确，较大分块则更通用但可能丢失细节。

我们已进行分块大小评估实验。

调整向量索引分块大小时，建议同步增加 similarity_top_k 参数以优化查询数据量：

from llama_index.core import SimpleDirectoryReader, VectorStoreIndex
from llama_index.core import Settings

documents = SimpleDirectoryReader("./data").load_data()

Settings.chunk_size = 512
Settings.chunk_overlap = 50

index = VectorStoreIndex.from_documents(
    documents,
)

query_engine = index.as_query_engine(similarity_top_k=4)

本例将默认分块大小减半，因此 similarity_top_k 也从默认值 2 调整为 4。

混合搜索#

混合搜索指结合语义搜索（嵌入相似度）与关键词搜索的检索方式。

当嵌入模型无法返回含匹配关键词的文本块时，混合搜索通常是解决方案。LlamaIndex 提供两种实现方式：

  1. 使用支持混合搜索的向量数据库（参见支持的向量存储列表）
  2. 通过 BM25 建立本地混合搜索机制

相关实现指南： - BM25 检索器 - 互逆重排查询融合 - Weaviate 混合搜索 - Pinecone 混合搜索 - Milvus 混合搜索

元数据过滤#

在文档存入向量索引前附加元数据非常有用。这些元数据既可用于追踪 response 对象答案来源，也能在查询时过滤数据：

from llama_index.core import VectorStoreIndex, Document
from llama_index.core.vector_stores import MetadataFilters, ExactMatchFilter

documents = [
    Document(text="text", metadata={"author": "LlamaIndex"}),
    Document(text="text", metadata={"author": "John Doe"}),
]

filters = MetadataFilters(
    filters=[ExactMatchFilter(key="author", value="John Doe")]
)

index = VectorStoreIndex.from_documents(documents)
query_engine = index.as_query_engine(filters=filters)

若使用 GPT-4 等高级 LLM 且向量数据库支持过滤，可通过 AutoVectorRetriever 实现查询时自动生成过滤器： - 向量存储指南

多租户 RAG#

多租户机制对 RAG 系统的数据安全至关重要，它能确保用户仅能访问自己的索引文档，防止未授权共享。搜索操作仅限用户自有数据，有效保护敏感信息。可通过 VectorStoreIndex 和 VectorDB 供应商的元数据过滤功能实现。

参考以下指南： - 多租户 RAG

Qdrant 发布的技术博客详细介绍了 LlamaIndex 与 Qdrant 的多租户实现方案。
