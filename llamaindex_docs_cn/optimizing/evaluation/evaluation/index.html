评估#

背景设定#

LlamaIndex 旨在将您的数据与 LLM 应用程序连接起来。

有时，即使通过查看跟踪记录诊断并修复了错误，仍需要更细粒度的评估来系统地诊断问题。

LlamaIndex 致力于提供这些工具，使识别问题和获取有用的诊断信号变得简单。

与评估紧密相关的是实验和实验跟踪的概念。

总体策略#

在开发 LLM 应用程序时，首先定义一个端到端的评估工作流程可能会有所帮助。一旦您开始收集失败或边缘案例，并对哪些方面运行良好或不佳有了直觉，您就可以更深入地评估和改进特定组件。

这与软件测试中的集成测试和单元测试类似。一旦开始调整单个组件，您可能应该开始编写单元测试。同样，判断事物是否协同工作的黄金标准是集成测试。两者同等重要。

端到端还是组件级——我该从哪个开始？#

如果您想在迭代过程中了解系统的整体表现，那么围绕端到端评估构建核心开发循环是有意义的——作为一种整体的健全性/氛围检查。

如果您清楚自己在做什么，并希望逐步迭代每个组件，随着进展逐步构建——您可能希望从组件级评估开始。然而，这可能会带来过早优化的风险——在没有评估整体应用程序需求的情况下做出模型选择或参数选择。在创建最终应用程序时，您可能需要重新审视这些选择。

深入评估#

评估是一个有争议的话题，随着 NLP 领域的发展，评估方法也在不断演变。

在强大的基础模型现在执行标注任务比人类标注者更好的世界里，关于评估的最佳实践在不断变化。过去用于引导和评估当今模型（如 BLEU 或 F1）的评估方法已被证明与人类判断的相关性较差，需要谨慎应用。

通常，生成密集、开放式的任务以及需要判断或意见的任务由于其主观性，比事实性问题更难自动评估。我们将致力于提供更多指南和案例研究，说明在特定场景中哪些方法是合适的。

标准指标#

针对标注数据集（无论是您自己的数据还是学术基准），有许多标准指标值得了解：

  1. 精确匹配（EM）： 完全正确回答的查询百分比。
  2. 召回率： 正确回答的查询百分比，不考虑返回的答案数量。
  3. 精确率： 正确回答的查询百分比除以返回的答案数量。
  4. F1 分数： F1 分数是精确率和召回率的调和平均数。因此，它在一个指标中对称地代表了精确率和召回率，同时考虑了假阳性和假阴性。

这篇 towardsdatascience 文章 更深入地介绍了 NDCG、MAP 和 MRR 等技术指标。
