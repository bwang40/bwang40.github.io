数据加载（摄取）#

在您选择的大语言模型（LLM）能够处理数据之前，首先需要对数据进行处理和加载。这与机器学习领域中的数据清洗/特征工程流程，或传统数据环境中的ETL流程类似。

数据摄取流程通常包含三个主要阶段：

  1. 加载数据
  2. 转换数据
  3. 索引和存储数据

我们将在后续章节中介绍索引/存储相关内容。本指南主要讨论加载器和数据转换。

加载器#

在LLM处理数据之前，您需要先加载数据。LlamaIndex通过数据连接器（也称为Reader）实现这一功能。数据连接器从不同数据源摄取数据，并将其格式化为Document对象。Document是数据（当前主要是文本，未来将支持图像和音频）及其元数据的集合。

使用SimpleDirectoryReader加载#

最简单的加载器是SimpleDirectoryReader，它会将给定目录中的每个文件转换为文档。该加载器内置于LlamaIndex，支持多种格式包括Markdown、PDF、Word文档、PowerPoint幻灯片、图像、音频和视频。

from llama_index.core import SimpleDirectoryReader

documents = SimpleDirectoryReader("./data").load_data()

使用LlamaHub中的加载器#

由于数据来源众多，并非所有连接器都内置。您可以从我们的数据连接器注册中心LlamaHub下载它们。

以下示例中，LlamaIndex下载并安装了DatabaseReader连接器，该连接器对SQL数据库执行查询并将结果每一行作为Document返回：

from llama_index.core import download_loader

from llama_index.readers.database import DatabaseReader

reader = DatabaseReader(
    scheme=os.getenv("DB_SCHEME"),
    host=os.getenv("DB_HOST"),
    port=os.getenv("DB_PORT"),
    user=os.getenv("DB_USER"),
    password=os.getenv("DB_PASS"),
    dbname=os.getenv("DB_NAME"),
)

query = "SELECT * FROM users"
documents = reader.load_data(query=query)

LlamaHub上有数百种可用连接器！

直接创建文档#

您也可以不使用加载器，直接创建Document对象。

from llama_index.core import Document

doc = Document(text="text")

数据转换#

数据加载后，在存入存储系统前需要进行处理和转换。这些转换包括分块、提取元数据以及为每个分块生成嵌入向量。这些步骤对于确保数据可被检索并被LLM高效使用至关重要。

转换的输入输出都是Node对象（Document是Node的子类）。转换步骤可以堆叠和重新排序。

我们提供高级和低级两套API来进行文档转换。

高级转换API#

索引对象有.from_documents()方法，可接受Document对象数组并自动进行解析和分块。但有时您可能需要更精细地控制文档分割方式。

from llama_index.core import VectorStoreIndex

vector_index = VectorStoreIndex.from_documents(documents)
vector_index.as_query_engine()

底层实现会将Document分割为Node对象，Node与Document类似（包含文本和元数据），但会保留与父Document的关系。

如果想自定义核心组件（如文本分割器），可以通过传入自定义的transformations列表或修改全局Settings实现：

from llama_index.core.node_parser import SentenceSplitter

text_splitter = SentenceSplitter(chunk_size=512, chunk_overlap=10)

# 全局设置
from llama_index.core import Settings

Settings.text_splitter = text_splitter

# 单索引设置
index = VectorStoreIndex.from_documents(
    documents, transformations=[text_splitter]
)

低级转换API#

您也可以显式定义这些步骤。

既可以单独使用我们的转换模块（文本分割器、元数据提取器等），也可以在我们的声明式转换管道接口中组合使用。

下面逐步说明：

将文档分割为节点#

处理文档的关键步骤是将其分割为"块"/Node对象。核心思想是将数据处理成适合检索/输入LLM的小块。

LlamaIndex支持多种文本分割器，包括基于段落/句子/标记的分割器，以及基于文件类型的分割器如HTML、JSON。

这些可以单独使用或作为摄取管道的一部分。

from llama_index.core import SimpleDirectoryReader
from llama_index.core.ingestion import IngestionPipeline
from llama_index.core.node_parser import TokenTextSplitter

documents = SimpleDirectoryReader("./data").load_data()

pipeline = IngestionPipeline(transformations=[TokenTextSplitter(), ...])

nodes = pipeline.run(documents=documents)

添加元数据#

您可以选择为文档和节点添加元数据。这可以手动完成，也可以使用自动元数据提取器。

以下是关于1)如何自定义文档和2)如何自定义节点的指南。

document = Document(
    text="text",
    metadata={"filename": "<doc_file_name>", "category": "<category>"},
)

添加嵌入向量#

要将节点插入向量索引，节点应包含嵌入向量。详见我们的摄取管道或嵌入指南。

直接创建和传递节点#

如果需要，您可以直接创建节点并将节点列表传递给索引器：

from llama_index.core.schema import TextNode

node1 = TextNode(text="<text_chunk>", id_="<node_id>")
node2 = TextNode(text="<text_chunk>", id_="<node_id>")

index = VectorStoreIndex([node1, node2])
