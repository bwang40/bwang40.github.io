索引#

数据加载完成后，您现在拥有一个文档对象列表（或节点列表）。接下来需要构建这些对象的Index索引，以便开始进行查询。

什么是索引？#

在LlamaIndex术语中，Index是由Document对象组成的数据结构，旨在支持LLM进行查询。索引的设计应与您的查询策略相辅相成。

LlamaIndex提供多种不同的索引类型。这里我们将介绍两种最常用的类型。

向量存储索引#

VectorStoreIndex是目前最常用的索引类型。向量存储索引会将您的文档分割成多个节点，然后为每个节点的文本创建向量嵌入，供LLM查询使用。

什么是嵌入？#

向量嵌入是LLM应用程序运行的核心机制。

向量嵌入（通常简称为嵌入）是文本语义或含义的数值化表示。即使实际文字差异很大，含义相似的两段文本在数学上会具有相似的嵌入表示。

这种数学关系实现了语义搜索功能：用户提供查询词时，LlamaIndex可以定位与查询词含义相关的文本，而非简单的关键词匹配。这是检索增强生成（RAG）技术的核心原理，也是LLM运作的重要基础。

存在多种嵌入类型，它们在效率、效果和计算成本上各有差异。LlamaIndex默认使用OpenAI的text-embedding-ada-002嵌入模型。如果使用不同的LLM，通常需要搭配不同的嵌入模型。

向量存储索引的嵌入过程#

向量存储索引通过LLM提供的API将所有文本转化为嵌入表示，这就是所谓的"文本嵌入"过程。如果文本量很大，生成嵌入可能非常耗时，因为这涉及大量API往返调用。

进行搜索时，您的查询语句也会被转化为向量嵌入，然后向量存储索引会执行数学运算，根据所有嵌入与查询语句的语义相似度进行排序。

Top K检索#

排序完成后，向量存储索引返回相似度最高的嵌入及其对应的文本片段。返回的嵌入数量由参数k决定，因此控制返回数量的参数被称为top_k。这类搜索常被称为"top-k语义检索"。

Top-k检索是向量索引查询的最基础形式，在查询章节您将学习更复杂精细的查询策略。

使用向量存储索引#

使用向量存储索引时，传入加载阶段创建的文档列表：

from llama_index.core import VectorStoreIndex

index = VectorStoreIndex.from_documents(documents)

Tip

from_documents方法接受可选参数show_progress。设置为True可在索引构建时显示进度条。

也可以直接基于节点列表构建索引：

from llama_index.core import VectorStoreIndex

index = VectorStoreIndex(nodes)

文本被索引后，理论上已准备好进行查询！但嵌入所有文本可能非常耗时，如果使用托管LLM服务还可能产生高昂费用。为了节省时间和成本，建议先存储嵌入。

摘要索引#

摘要索引是一种更简单的索引类型，特别适合（正如其名）需要生成文档摘要的查询场景。它只是存储所有文档，并在查询时将所有文档返回给查询引擎。
